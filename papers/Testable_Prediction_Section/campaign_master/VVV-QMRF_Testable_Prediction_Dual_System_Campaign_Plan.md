## 1. Campaign Purpose

The Testable Prediction Campaign separates two needs that should not be compressed into one paper system:

1. **Internal system:** full technical development, RCA, source traceability, claim registry, proof gaps, and internal digital evidence.
2. **External system:** public-facing papers with built-in disclaimer, conservative claim language, minimal notation, and public critique evidence.

The campaign's root question is:

```text
Interaction X có phải là một valid registered measurement không?

In English:
Does interaction X count as a valid registered measurement for registering process R?
```

The campaign develops this question in stages:

```text
ValidReg(X, R)
→ ValidReg failure/control cases
→ K-side registration spaces
→ K_joint embedding problem
→ Extended Wigner's Friend stress test
→ K-side incommensurability conjecture
→ experimental disconfirmation target
→ interpretation positioning
```

## 2. RCA Summary

### 2.1 Phenomenon

VVV-QMRF needs a credible route from registration-layer theory toward testability without presenting an unverified physical replacement for Standard Quantum Mechanics.

### 2.2 Proximate Cause

A single paper would have to carry incompatible functions: full internal proof-control and public-facing readability.

### 2.3 Underlying Mechanism

Internal work requires RCA, source hierarchy, claim classes, symbol registries, proof gaps, and digital evidence. External work requires concise framing, scoped claims, and readable public argument.

### 2.4 Root Cause / New Principle

The research-control layer and public-communication layer must be separated while preserving a one-to-one trace between them.

### 2.5 Generalization

Every campaign paper should exist as an internal technical version and, when ready, an external public-facing version.

## 7. External System Contract

External papers must include:

```text
Built-in disclaimer
Clear research question
Conservative thesis
Minimal notation
Short source note
Readable argument flow
Public evidence note
Invitation for critique
```

External papers must not include:

```text
Long RCA process sections
LLM workflow notes
Self-awarded decision scores
Internal proof bureaucracy
Unverified citations
Claims of experimental validation not supported by external evidence
```

## 9. Source and Boundary Rules

1. If a claim uses Buddhist Epistemology node or edge definitions, verify against `SYSTEM_Buddhist_Epistemology/system_be_full.md`.
2. If a claim uses VVV-QMRF symbols, verify against the active framework or meta-architecture source.
3. If a claim touches physical quantum mechanics, preserve Standard QM at the physical `ρ`-side.
4. If a claim uses `K_joint`, `⊥_K`, or `M′ ⊥ M`, classify it as Class C unless a formal definition and proof are supplied.
5. If a formula appears stronger than the source support, downgrade the claim class.
6. External papers must translate claim classes into readable boundary language.
